# Challenge 06: POS Log Parser

## Problem

A Point-of-Sale (POS) system generates log files with a custom format. You must parse these logs and produce a structured report.

## Log Format

Each line follows this format:
```
[YYYY-MM-DD HH:MM:SS] EVENT_TYPE | field1=value1 | field2=value2 | ...
```

Event types:
- `ORDER_PLACED` — `order_id=N | items=N | total=N.NN`
- `ORDER_COMPLETED` — `order_id=N | duration_secs=N`
- `ITEM_SOLD` — `item_name=NAME | quantity=N | price=N.NN | order_id=N`
- `PAYMENT` — `order_id=N | method=CARD/CASH/MOBILE | amount=N.NN`

## Task

Edit `solution.py` to implement `parse_logs(log_file_path)` that reads the log file and returns a formatted report string matching the format in `expected_output.txt`.

The report must include:
1. **Hourly Revenue Table** — Revenue broken down by hour (HH:00 format)
2. **Top 10 Items** — By total quantity sold
3. **Average Ticket Time** — Mean time between ORDER_PLACED and ORDER_COMPLETED

## Setup

```bash
python generate_logs.py   # Creates pos_logs.txt and expected_output.txt
python -m pytest test_logparse.py -v
```

## Files

- `generate_logs.py` — Generates log data. Run this first.
- `solution.py` — Your solution. Edit this file.
- `test_logparse.py` — Tests. Do NOT modify.
- `expected_output.txt` — Expected output (generated by generate_logs.py).
